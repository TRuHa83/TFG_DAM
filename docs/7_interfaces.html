7. Diseño de Interfaces (UI/UX)

El desarrollo de la Interfaz Gráfica de Usuario (GUI) para Dotfiles-Manager se encuentra actualmente en fase de conceptualización (Wireframing). Las vistas y funcionalidades descritas a continuación representan el planteamiento inicial y están sujetas a evolución durante la fase de desarrollo e implementación con el framework PySide6.

7.1. Patrón de Navegación

Para maximizar el espacio de trabajo horizontal —crítico al visualizar rutas absolutas de archivos en sistemas Linux— la aplicación adoptará un layout basado en un Menú de Navegación Lateral (Sidebar). Este menú agrupará las funcionalidades en pestañas verticales con iconos descriptivos, cargando el contenido dinámicamente en el área principal.

7.2. Desglose Conceptual de Vistas

1. Telemetría y Resumen

Pantalla de aterrizaje principal diseñada para ofrecer una visión global del estado del sistema al iniciar la aplicación.

Dashboard
  • Métricas de escaneo: Número total de dotfiles detectados en el sistema.
  • Estado de clasificación: Ratio exacto de archivos clasificados frente a archivos sin clasificar.
  • Volumen de software: Conteo total de aplicaciones soportadas y detectadas en la máquina.
  • Información del sistema: Versión actual del software e información básica relevante (ej. fecha del último escaneo).

2. Gestión de Inventario

El núcleo interactivo de la aplicación, donde el usuario audita y manipula las relaciones entre los dotfiles y su software correspondiente.

Aplicaciones Listadas

Catálogo visual del software reconocido, acompañado de sus iconos representativos.

  • Visualización en detalle de los archivos vinculados a cada aplicación.
  • Capacidad para editar los metadatos de la aplicación.
  • Acciones directas para vincular rutas adicionales o desvincular archivos asignados erróneamente.
Archivos Desconocidos

Listado de las rutas que el motor heurístico local no ha logrado identificar de forma automática.

  • Controles para agregar o eliminar estos archivos de las listas de ignorados del sistema.
  • Opción para crear una aplicación nueva manualmente y vincularle estos archivos.
  • Acción específica para marcar archivos y solicitar su identificación vía LLM (activando la conexión con el servidor).

3. Administración y Mantenimiento

Módulo destinado a la configuración del comportamiento interno de la herramienta.

Ajustes
  • Configuración de parámetros básicos de la interfaz y la aplicación.
  • Gestor de reglas de ignorados (añadir/eliminar exclusiones personalizadas).
  • Módulo para exportar/importar ajustes globales (facilitando la migración a otros equipos).
  • Opciones de mantenimiento, como el reseteo completo de la base de datos local (DB Wipe).

Nota: El resto de funcionalidades avanzadas propuestas en los objetivos (como la configuración de Tareas Cron o la gestión de Backups) se diseñarán e integrarán en el menú lateral de forma progresiva a medida que avance el desarrollo.
