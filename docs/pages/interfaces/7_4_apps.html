7.4. Vista: Aplicaciones

La vista de Aplicaciones es el núcleo interactivo del inventario. Muestra el catálogo completo del software reconocido por la herramienta: cada aplicación con sus metadatos (nombre, categoría, icono si está disponible) y la lista de rutas de dotfiles que tiene asociadas en el sistema local.

Figura 7: Vista Aplicaciones.

Listado de Aplicaciones Clasificadas

El panel principal muestra todas las entradas del inventario local (local_inventory) agrupadas por aplicación. Para cada aplicación se presenta:

Información de la Aplicación
  • Nombre legible y categoría extraídos de known_apps_reference.
  • Icono representativo cuando esté disponible en los recursos de la aplicación.
  • Lista de dotfiles vinculados: rutas absolutas de cada archivo o directorio de configuración asociado a la aplicación en el equipo del usuario.

Acciones Disponibles

Gestión del inventario
  • Vincular ruta (bind): permite al usuario asociar manualmente una ruta adicional de su sistema a una aplicación existente.
  • Desvincular (unbind): elimina la asociación entre una ruta y su aplicación, devolviéndola al grupo de desconocidos o eliminándola del inventario.
  • Editar metadatos: modificar el nombre legible o la categoría de la aplicación en la base de datos local.

Nota: Las acciones de gestión del inventario desde esta vista están planificadas para la siguiente iteración del desarrollo. La visualización del catálogo ya es funcional.
