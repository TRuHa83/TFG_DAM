4.4.1 Ejecución en Hilo Independiente (QThread)

Cuando el usuario interactúa con la interfaz gráfica (UI), las operaciones de pipeline pueden implicar lecturas intensivas del sistema de archivos o consultas a la base de datos. Ejecutar estas operaciones directamente en el hilo principal de Qt bloquearía la interfaz, dejándola sin respuesta durante el tiempo de procesamiento.

Para evitar este bloqueo, la aplicación implementa el patrón Worker / QThread: cada vez que el usuario lanza una operación (por ejemplo, pulsando el botón main_check), el método run_pipeline() de MainWindow instancia un objeto Worker(QThread) y lo inicia en un hilo secundario.

UI (hilo principal) → Worker(QThread) (hilo secundario)
    Worker.run() → Pipeline → Steps → Reporter
    Reporter → Handler.método(context)
    Handler → Signal Qt → MainWindow (hilo principal)
Figura 3: Flujo de comunicación entre hilos durante la ejecución de un pipeline desde la UI.

Las señales Qt (ui_signal, update_widget_signal) son el mecanismo seguro que provee PySide6 para cruzar datos desde un hilo secundario hacia el hilo principal de la UI. De esta forma, los widgets de la interfaz se actualizan de forma completamente asíncrona.
